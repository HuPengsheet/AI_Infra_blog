# vLLM 与 KV Cache 速记

在大模型在线推理中，KV Cache 的意义几乎等于“把重复计算提前变成可复用状态”。

## 为什么它重要

对于自回归生成模型来说，已经生成过的 token 不需要在每一步都重新计算全部注意力状态。KV Cache 让模型可以直接复用历史 key/value，从而：

- 降低单步生成开销；
- 提升整体吞吐；
- 改善长上下文场景下的时延表现。

## 实践里要看的指标

- 首 token 时延：模型加载、prefill、调度的综合结果
- 每 token 生成速度：decode 阶段的核心指标
- 显存占用：上下文长度和并发一起决定压力
- 批处理效率：连续批调度能否吃满 GPU

## 常见工程权衡

### 1. 吞吐和时延不是同一个目标

更大的 batch 往往提升吞吐，但也可能让单请求排队时间变长。

### 2. 上下文越长，缓存管理越关键

长上下文带来更高显存占用，缓存淘汰策略、分页管理和并发分桶都会影响稳定性。

### 3. 压测要区分 prefill 和 decode

如果只看平均响应时间，很容易掩盖真实瓶颈。更合理的方式是分别观察：

- prompt token 数量变化时的 prefill 耗时
- output token 数量变化时的 decode 耗时

## 一条经验结论

如果你的业务已经进入“稳定在线推理”阶段，优先关注的通常不是再换一个模型框架，而是：

1. 调度是否合理；
2. 缓存是否高效；
3. 指标是否足够细；
4. 回归对比是否自动化。
